1 Project Overview

This project is not a traditional codebase. There are no scripts, no application server, no package.json. The entire "engine" is a set of procedure documents (Markdown) and JSON state files that Claude Code reads and follows as instructions, using MCP tools to interact with external services.

What It Does

Runs iterative optimization cycles where a "champion" landing page competes against "challenger" variants. Traffic is split via RedTrack, performance is measured by CVR/EPC, and winners are promoted using chi-squared statistical testing. Losers are discarded. New challengers are generated each cycle.

Current Vertical

Ecommerce — Olenis "Bright Eyes" Peptide Eye Patches ($23 product)

Traffic Source

Facebook Ads → RedTrack → 3 lander variants (34/33/33 split)

Champion

"7 Reasons Why..." listicle on Framer (landing.olenis.com). Baseline CVR: 3.53%

Challengers

Nutra editorial (4 components changed) + Scarcity angle (2 components changed) on Netlify

The Learning Loop

Each cycle teaches the system something. Winning component tags accumulate win records. After 20+ cycles, pattern detection activates to find repeatable formulas (e.g., "urgency + scarcity = +15% CVR").

4 Key Workflows

Optimization Cycle (core loop)

Defined in engine/cycle-procedure.md. This is the main process Claude follows when told "Run optimization cycle for [vertical]".

  1. 1

    Load State

    Read champions.json, cycles.json, verticals.json, redtrack-campaigns.json

  2. 2

    Pull Performance Stats

    Call rt_get_stats grouped by landing. Check: min 100 clicks/variant AND 48 hours elapsed.

  3. 3

    Determine Winner

    Chi-squared test on CVR. X² > 3.841 = p<0.05 = statistically significant. 3-way pairwise comparisons.

  4. 4

    Notify via Telegram

    Send formatted results + grading prompt to Wesley via tg_send_message.

  5. 5

    Wait for Human Grading

    Read Wesley's tag attribution from Telegram. Which component tags drove the win?

  6. 6

    Update Learning Data

    Write to component-wins.json (tag records), champions.json (promote winner), cycles.json (complete).

  7. 7

    Generate New Challengers

    Build HTML manually for ecommerce (presales doesn't support it). Change 1-3 components, tag each change.

  8. 8

    Update RedTrack Campaign

    Set 34/33/33 traffic split across champion + 2 new challengers via RT API.

  9. 9

    Log & Confirm

    Write new cycle to cycles.json. Send Telegram notification with next check time.

Click ID Passthrough (critical for conversion tracking)

// 1. RT trackback URL redirects user to lander with ?sub2={clickid}
// 2. Lander JavaScript captures sub2 param:
var clickid = params.get('sub2') || params.get('clickid');
if (clickid) {
    document.querySelectorAll('.cta-link').forEach(function(link) {
        var url = new URL(link.href);
        url.searchParams.set('sub2', clickid);
        link.href = url.toString();
    });
}
// 3. CTA click → Shopify with sub2 → Shopify postback to RT with clickid

8 Configuration

Central campaign config. Contains RT campaign ID, offer ID, lander IDs (champion + challengers), baseline performance metrics, and traffic source info.

  • Significance thresholds (min 100 clicks, 48 hours)
  • CVR targets per vertical (excellent / good / flag)
  • Component tag libraries per component type
  • Product context (brand, store domain, offer IDs)
